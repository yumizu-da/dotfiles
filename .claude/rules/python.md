# Pythonコーディング規約

## 基本方針

- Python 3.12+ 時代における「Pythonic」なコードを書くことを基本とする
- [Google Python Style Guide](https://google.github.io/styleguide/pyguide.html) に準拠し、必要に応じて以下の指示やプロジェクトのニーズを優先する
- 可読性、保守性、そしてパフォーマンスを重視した現代的な開発基準として機能することを目指す

## 哲学

- 読みやすさが第一: コードは書かれる回数より読まれる回数の方が多い
- シンプル > 複雑: 高度な機能（メタクラス、動的継承、`getattr` の悪用など）は避ける
- 明示的 > 暗黙的: 暗黙の挙動に頼らない
- 可変グローバル状態を避ける
- イミュータビリティ: 既存オブジェクトを変更せず新しいオブジェクトを作成

## 型ヒントと静的解析

- すべての関数引数と戻り値に型ヒントを記述する
- 標準コレクション型（`list`, `dict`, `tuple`, `X | None` 等）での型ヒントを使用する。`typing` から `Optional`, `Dict`, `List`, `Tuple` をインポートしない
- 命名規則: 変数/関数=snake_case、クラス=PascalCase、定数=UPPER_SNAKE_CASE
- 返り値が自身のインスタンスである場合、`typing.Self` を使用する

## イディオム

- ネストを深くせず、早期離脱を心がける
- `None` はシングルトンなので、`is None` / `is not None` を使う
- ファイル・リソース操作は `with` ステートメントを使う
- パス操作は `os.path` より `pathlib` を優先する（`from pathlib import Path` でインポートして使う）
- 文字列フォーマットは f-string を優先する（`%` や `.format()` より）
- `def func(a=[])` は禁止。`def func(a=None)` とし内部で初期化する
- 内包表記は、`map` や `filter` よりもリスト/辞書内包表記を優先する。ただし複雑すぎる場合は可読性が低くなるので、通常のループにする
- 基本的には絶対インポートを使う（ただし、既存コードで既に相対インポートが使用されている場合は除く）

## 過度な関数化を避ける

簡単な式や1度しか使わない自明な処理を関数でラップしない。関数を作る前に、まず呼び出し側にそのまま書けないかを検討する。

関数化してよいのは、以下のいずれかを満たす場合のみ。

- 2箇所以上で使われる
- 名前を付けることで意図が明確になる（例: `is_valid_email`）
- 処理が複雑で、抽象化することで読みやすくなる

### 悪い例

```python
def _check_hoge(a: int) -> bool:
    return a > 0

if _check_hoge(value):
    ...
```

上記の例では、`if value > 0:` とそのまま書けばよい。

## エラーハンドリング

- `except Exception` や bare `except:` で広く捕まえない。特定の例外だけ捕捉する
- `try` ブロックは必要最小限にする（想定外の例外を巻き込まない）
- 捕捉したら、ログ出力するか再raiseする。黙って `pass` や握りつぶしをしない

## Docstring

- 型ヒントで自明な情報は書かない。型ヒントだけでは伝わらない情報（引数の意味、関数の挙動、アルゴリズム、副作用など）だけを記述する
- ただしプロジェクトのスタイルガイドで、型ヒントと冗長な説明を許容する場合はこの限りではない

### 良い例: 型ヒントと簡潔な説明

```python
def add(a: int, b: int) -> int:
    """2つの整数を加算して返す"""
    return a + b
```

### 良い例: 型ヒントだけでは不明確な情報がある場合

```python
def calculate_distance(lat1: float, lon1: float, lat2: float, lon2: float) -> float:
    """2地点間の距離を計算

    Haversine公式を使用して球面上の最短距離を算出。

    Args:
        lat1, lon1: 始点の緯度・経度（度数法）
        lat2, lon2: 終点の緯度・経度（度数法）

    Returns:
        距離（キロメートル）
    """
```

### 悪い例: 型ヒントと冗長な説明

```python
def add(a: int, b: int) -> int:
    """2つの整数 a と b を加算して整数を返す

    Args:
        a (int): 最初の整数
        b (int): 2番目の整数

    Returns:
        int: 2つの整数の和
    """
```
